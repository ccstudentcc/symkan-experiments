# Code of Conduct

## 行为原则

本项目致力于维持直接、专业且可持续的协作环境。讨论可以严格，判断可以尖锐，但表达应始终围绕技术问题与协作目标展开。

以下行为符合本项目的协作预期：

- 基于事实、证据与复现结果讨论问题。
- 对代码、设计与实验结论提出具体且可执行的反馈。
- 尊重不同经验水平与研究背景的参与者。
- 在指出问题时同时说明其影响范围与可能的改进方向。
- 在方案存在分歧时继续围绕问题本身沟通，而非转向个人评价。

以下行为不可接受：

- 人身攻击、羞辱、嘲讽、骚扰或持续性敌对表达。
- 带有歧视、仇恨、威胁性质的言论。
- 故意破坏讨论秩序、制造对立或进行恶意刷屏。
- 泄露他人隐私、凭证、密钥或未公开数据。
- 明知会破坏既有工作流仍在缺乏讨论与说明的情况下强行推进相关改动。

## 适用范围

本准则适用于以下协作场景：

- issue
- pull request
- 代码审查评论
- 与本项目相关的讨论区或协作渠道

## 处理方式

维护者可以根据具体情况采取以下措施：

- 删除不当内容；
- 拒绝相关贡献；
- 暂时或永久限制参与。

处理的首要目标是保护协作环境与项目运行秩序。

## 报告方式

若遇到骚扰、恶意行为或不当披露，请联系项目维护者，并提供必要的上下文信息。

如问题涉及安全漏洞、密钥泄露或其他敏感内容，请优先遵循 [SECURITY.md](SECURITY.md) 中的披露流程，而不要直接公开细节。
