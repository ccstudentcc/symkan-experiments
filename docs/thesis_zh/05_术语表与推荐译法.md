## 1. 方法与流程相关

| 英文原词 | 推荐中文 | 说明 |
| --- | --- | --- |
| post-training KAN symbolization | 训练后 KAN 符号化 | 建议作为主译法，不再来回切换成“后处理符号回归” |
| stagewise_train | 分阶段训练 | 保留代码名时用反引号，正文解释时用中文 |
| shared symbolic-prep | 共享符号预处理 | 用于说明 backend compare 之前共享的阶段 |
| backend-specific symbolic completion | 后端特异的符号完成 | 用来指代 `baseline` 与 `icbr` 真正分叉的环节 |
| backend-only compare | 仅后端差异比较 | 强调差异只发生在后端环节 |

## 2. 结果与证据相关

| 英文原词 | 推荐中文 | 说明 |
| --- | --- | --- |
| evidence hierarchy | 证据层级 | 比“材料分组”更准确 |
| fairness gate | 公平性门槛 | 只在说明 compare 前提时使用 |
| paired compare | 配对对比 | 用于 shared-state 已对齐的两条变体 |
| supplementary single-arm slice | 补充性单边切片 | 适合描述 `baseline_icbr_fulllib` |
| structured export | 结构化导出 | 对应 CSV、JSON、trace 等结果产物 |

## 3. 指标相关

| 指标名 | 推荐中文写法 | 备注 |
| --- | --- | --- |
| `symbolic_core_seconds` | 核心符号阶段耗时 | backend 速度主指标优先用它 |
| `symbolize_wall_time_s` | 符号阶段墙钟时间 | 可报告，但不替代核心耗时 |
| `validation_mean_r2` | 公式验证均值 R² | 更接近导出公式与模型输出的一致性 |
| `formula_export_success_rate` | 公式导出成功率 | 不能改写成真实公式恢复成功率 |

## 4. 建议固定的几组短语

- “工程化符号实验流程”
- “共享符号预处理”
- “仅后端差异比较”
- “补充性单边切片”
- “结构化结果归档”

## 5. 不建议反复切换的说法

- “符号化”与“符号回归”不要无条件互换。当前仓库更常处理的是训练后 KAN 的符号化。
- “fairness” 不建议直译成“公平性”后单独悬空使用，最好和“对齐条件”或“比较边界”一起出现。
- “ICBR 更快”最好落到“核心符号阶段提速”这种更窄的说法上。
