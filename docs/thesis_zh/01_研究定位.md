## 1. 本仓库在论文中的角色

`symkan-experiments` 更适合被写成本文的主实验仓库。它关心的不是重新发明 KAN 数值训练器，而是在保留既有训练语义的前提下，把训练后的 KAN 推进到可符号化、可验证、可比较、可归档的状态。

从论文表述上看，这个仓库更接近“面向训练后 KAN 的工程化符号实验框架”，而不是“从原始数据直接做通用符号回归的全新算法”。这一点与 `docs/design.md` 和 `docs/symkan_manuscript.md` 的主线是一致的。

## 2. 可以直接承接到绪论或方法章的定位句

可直接使用的表述有三类。

第一类是研究对象的界定。本文关注训练后 KAN 的符号化问题，即在既有数值模型已经形成之后，如何通过阶段化准备、符号后端求解和结果验证，把数值模型转成可检查的符号表达式。

第二类是工程目标的界定。本文工作的重点不是单次精度，而是配置统一、流程分段、比较公平性和结果可追溯性。

第三类是贡献边界的界定。本文更稳妥的贡献写法，是提出一套围绕训练后 KAN 的工程化符号实验流程，并通过结构化证据说明默认流水线为何成立、ICBR 后端在什么条件下可以被解释为 backend-only 收益。

## 3. 不宜写成本文贡献的内容

- 不宜把 SymKAN 写成新的 KAN 数值训练算法。
- 不宜把 SymKAN 写成从原始样本直接恢复真实闭式公式的通用符号回归系统。
- 不宜把 `n=3` 的工程复测写成充分的统计显著性结论。
- 不宜把 `formula_export_success_rate=1.0` 改写成“真实公式恢复成功率为 100%”。

## 4. 与其他仓库的关系

- `KAN-Symbolic_Regression` 更适合作为统一对比实验和扩展案例的支撑材料。
- `ICBR-KAN` 是另一条由你维护的独立实验线，可单独成章，但不替代本仓库的主实验主线。
- `SymbolNet-main - 副本` 与 `ICBR-KAN/In-Context-Symbolic-Regression-KAN-main` 只适合作为 comparison-only 的外部方法或复现包来源。
