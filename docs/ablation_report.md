# symkan 单点消融实验报告

**实验日期**：2026-03-14  
**数据集**：内部 MNIST 特征集（10类，784维输入）  
**实验设定**：seeds 42 / 52 / 62（n=3），global-seed=123，lib-preset=layered，target-edges=90

## 1. 实验设计回顾

本实验基于 `docs/ablation_plan.md` 的框架，对 symkan 流水线的四个核心设计模块做单点消融：

| 变体 ID | 变体名称 | 关闭的模块 | 关键 CLI 开关 |

## 3. 各模块单点消融分析

### 3.1 V1：去掉分阶段训练

**结论：分阶段训练是整个流水线最关键的基础组件，去掉后系统退化极其明显。**

最终精度从 0.777 暴跌至 0.442（**−33.5pp**），Macro-AUC 下降 11.9pp，符号化获得的有效输入维数从 57 降至 23。

根本原因已由 `pre_symbolic_too_dense` 标志验证：三次 seed 的该标志值均为 `True`（均值 = 1.0），模型进入符号化时的边数为 **2080**，是 stagewise 训练目标边数 120 的约 17 倍（符号化目标边数 90 的约 23 倍）。虽然 `--prune-collapse-floor 0.0` 关闭了回滚保护，渐进剪枝仍然将 2080 条边压缩到 `effective_target_edges=1040`（逻辑上是目标×输入维数的组合下限），但符号化面对如此稠密的图，能覆盖的有效维度很少，公式质量极差（R² = −0.814）。

**额外观察**：阶段训练耗时仅 13.7s（比基线快 65%），这是因为 e2e 一次性训练比多轮 stagewise 循环快得多——但省下的时间代价是完全无用的符号化输出。

---

### 3.2 V2：去掉渐进剪枝

**结论：去掉渐进剪枝对精度几乎无影响，但公式复杂度显著膨胀、符号化耗时增加。**

最终精度 0.775 与基线 0.777 差异在统计误差内（−0.16pp）。Macro-AUC 反而略升至 0.9595（+0.55pp），三种 seed 结果基线方差较大（std=0.034 vs 0.021），说明去掉剪枝后精度稳定性有所下降。

代价是明显的：

- 表达式平均复杂度从 130.8 飙升至 **217.1**（+66%），每个公式变得更难以阅读和解释；
- 有效输入维数从 57.0 上升到 74.3（+30%），更多冗余激活函数被强行符号化；
- 符号化阶段耗时从 51.2s 增加到 **67.8s**（+32%）。

R² 从 −0.632 改善至 −0.475，说明保留更多边对公式数值验证有些许帮助，但总体仍处于负值区间，不能实际使用。

---

### 3.3 V3：去掉输入压缩

**结论：去掉输入压缩对分类精度几乎无影响，但公式数值质量（R²）出现显著改善；代价是符号化耗时增加 51%。**

最终精度 0.769 与基线 0.777 差 0.8pp，Macro-AUC 几乎不变（0.9532 vs 0.9540）。

最显著的差异出现在公式验证 R² 上：基线为 −0.632，去掉输入压缩后提升至 **+0.096**（正值），这是五个变体中唯一整体 R² 为正的情况。直觉上：输入压缩会选取归因排名靠前的子集（57 维），但部分对分类有辅助作用的低权重维度也会被丢弃；保留全部 120 维输入后，公式在数值上能更准确地重拟合数据，尽管表达式本身不会更短。

另一个附带效应是符号化耗时从 51.2s 增加至 **77.2s**（+51%），直接因 effective_input_dim 从 57 翻倍到 120 导致。

---

### 3.4 V4：去掉层间微调

**结论：去掉层间微调不仅不降低精度，反而使精度方差缩小 15×、符号化时间减少 60%。层间微调是当前流水线中收益/代价比最差的模块。**

最终精度 0.784 略高于基线 0.777（+0.7pp，在 std 误差范围内）；Macro-AUC 几乎相同（0.9544 vs 0.9540）；表达式复杂度与有效输入维数均与基线一致。10 个类中有 6 个类的公式验证 R² 在去掉层间微调后反而升高，整体 R² 均值从 −0.632 改善至 −0.594。

精度方差显著收窄：std 从 **0.0210** 降至 **0.0014**（降低约 93%）。符号化耗时从 51.2s 缩短至 **20.5s**（−60%），额外节省约 30.7s 的层间微调时间对最终结果没有任何贡献。

详细对比数据由 `analyze_layerwiseft.py` 生成，见 `benchmark_ablation/layerwiseft_analysis/`。

### 3.4.1 层间微调失效的深度分析

#### 从 KAN 理论看

KAN 的表示依赖 Kolmogorov-Arnold 分解：$f(x) = \sum_q \Phi_q\bigl(\sum_p \phi_{qp}(x_p)\bigr)$。逐层符号化将第 $l$ 层的 B-spline 替换为解析函数 $a \cdot g(bx+c)+d$，这是有损替换——仿射参数 $(a,b,c,d)$ 由最小二乘拟合确定后被冻结。

层间微调的原始设计意图是：固定第 $l$ 层解析函数后，让第 $l+1$ 层通过梯度下降补偿第 $l$ 层引入的拟合误差。在深层 KAN（$\geq 3$ 层）中这可能有用；但在本实验的 **2 层 KAN**（`depth=2`）中，层间微调只在 `layer 0` 符号化后触发一次（代码条件 `l < depth - 1`），直接影响唯一的输出层。

#### 从 pykan 符号化实现看

`fix_symbolic` 使用激活样本做最小二乘拟合，基函数形状一旦确定就被冻结，后续全局微调只能调整仿射缩放，**不能改变已选择的基函数**。

层间微调形成了"贪心序列偏差"：

1. 已固定的 layer 0 解析函数决定了 layer 1 接收的输入分布
2. 200 步 Adam（`lamb=0.0`，无正则化）将 layer 1 的 B-spline 参数推向适应 layer 0 误差的局部极值
3. 当 layer 1 随后被符号化时，其基函数是对**过拟合分布**的拟合，而非对原始数据的最优拟合

这与 Deep Belief Net 中贪心逐层预训练的已知问题相同：局部最优叠加不等于全局最优。

#### 从过拟合与方差看

层间微调使用 `lamb=0.0`（无稀疏正则）、200 步 Adam、**无验证集早停**。在 B-spline 参数量较多时这等价于无约束的训练集微调，结果高度依赖 seed 的随机游走方向，造成精度 std 扩大 15×。全局 affine 重微调（600 步）在所有层符号化后统一执行，其优化空间更完整，能更好地恢复层间误差，因此不需要中间的层间微调。

#### 四假设验证结果（`analyze_layerwiseft.py` 输出）

| 假设 | 内容 | 验证结论 |
|------|------|---------|
| H1 | 全局重微调已覆盖层间调整 | ✓ 支持：\|Δacc\| = 0.0069 < 1pp |
| H2 | 层间微调引入贪心偏差 | ✓ 支持：6/10 类的 woft R² ≥ full R² |
| H3 | 无正则化层间微调增大方差 | ✓ 支持：full.std = 0.0210 vs woft.std = 0.0014 |
| H4 | 耗时与收益严重不匹配 | ✓ 支持：额外 30.7s，精度收益 −0.0069pp |

#### 改进建议

1. **引入验证集早停**：类似 `stagewise_train` 的 `--use-validation`，在 R² 不再提升时终止层间微调
2. **加入轻量正则化**：`lamb=1e-5`，防止 B-spline 过拟合而不破坏稀疏性
3. **缩短步数**：从 200 步减至 50~100 步，压缩过拟合窗口
4. **最终建议**：对于 2 层 KAN，直接关闭层间微调（`--layerwise-finetune-steps 0`），依赖全局 affine 重微调，在精度等价的前提下节省 60% 符号化时间

## 4. 综合分析

### 4.1 各模块对精度/可解释性/效率三维的贡献矩阵

| 模块 | 分类精度贡献 | 公式复杂度 | 公式数值质量（R²） | 计算效率 |
|------|------------|-----------|-----------------|--------|
| 分阶段训练 | **极关键** ↑33.5pp | 使公式更复杂 | 略有帮助 | 增加训练时间 |
| 渐进剪枝 | 不显著 | **压缩复杂度−40%** | 轻微负向 | 减少符号化时间 |
| 输入压缩 | 不显著 | 略减（−11%） | **明显负向（R² −0.73）** | 减少符号化时间 |
| 层间微调 | 不显著 | 基本无影响 | 略有帮助 | **最大耗时模块（占60%）** |

### 4.2 发现与设计权衡

**发现 1（分阶段训练是必要前提）**：没有 stagewise 训练，整个流水线无法从稀疏状态进入符号化，其他三个机制形同虚设。这印证了两段式架构的核心假设：符号化的成败首先取决于训练阶段能否给出足够稀疏的模型。

**发现 2（渐进剪枝是"可解释性-精度"的体积控制阀）**：它不提升精度，但将表达式体积压缩约 40%，直接影响公式的可读性。如果目标是产出短公式供人阅读，渐进剪枝不可省；如果仅追求精度，可以放宽或关闭。

**发现 3（输入压缩存在精度-验证质量权衡）**：输入压缩在保持分类精度的同时，**牺牲了公式的数值验证质量**（R² 从正转负）。原因可能在于被丢弃的低归因维度对公式重拟合仍有贡献。未来可以考虑引入更保守的压缩策略（如保留归因前 80% 累计权重而非 Top-K 维），或把压缩评分与公式验证 R² 联合优化。

**发现 4（层间微调是效率瓶颈，收益-代价比最差）**：层间微调消耗了符号化阶段约 60% 的时间，不仅没有系统性提升，反而将精度跨 seed 标准差扩大了约 15×（full std=0.021 vs woft std=0.001），说明其作用是引入不稳定性而非可靠收益。根本原因见 §3.4.1：无正则、无早停的 200 步 Adam 将输出层 B-spline 推入局部极小，导致后续符号拟合基函数选择质量下降。在计算资源有限或追求快速迭代的场景下，可以安全地以 `--layerwise-finetune-steps 0` 关闭，时间从 ~51s 降到 ~20s，精度保持不变且更稳定。

---

## 5. 结论

| 优先级 | 模块 | 建议 |
|--------|------|------|
| 必须保留 | 分阶段训练 | 是整个架构的基础，去掉则输出无效 |
| 建议保留 | 渐进剪枝 | 是公式可读性的核心来源，如无特殊需求不应关闭 |
| 场景取舍 | 输入压缩 | 精度无显著影响，但公式数值验证质量下降；加速重要时保留，公式数值质量重要时可关闭 |
| 可选关闭 | 层间微调 | 精度收益不显著，关闭后节省 60% 符号化时间；快速实验时建议关闭 |
